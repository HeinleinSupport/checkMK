# Reject non-UTF-8 input from Nagios command pipe

key        | value
---------- | ---
date       | 2026-02-19T08:32:05+00:00
version    | 2.6.0b1
class      | fix
edition    | pro
component  | cmc
level      | 1
compatible | yes

Invalid non-UTF-8 input from the Nagios command pipe can result in a corrupt
CMC state file. [Werk #14949](https://checkmk.com/werk/14949) already fixed
a similar issue for Livestatus input or check output, but missed the
validation of the Nagios command pipe input. This has been fixed.
