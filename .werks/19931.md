# Clarify purpose of the Microsoft Entra ID connections page

Previously, it was not obvious what the Setup page _Microsoft Entra ID
connections_ is for. Some users expected it to configure a general-purpose
identity provider or single sign-on, when in fact it serves a much narrower
purpose.

The page now shows an informational note explaining that a connection
configured here only creates an OAuth2 token used by the email-related active
checks (_Check email_, _Check email delivery_ and _Check IMAP/EWS/GraphApi
mailboxes_) to authenticate against Microsoft 365. It is not a general-purpose
identity provider or single sign-on connection.
